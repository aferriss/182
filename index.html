Overview

This course is an introduction to writing code in the context of the visual arts and design. The concepts taught in this course will provide a foundation for future courses in generative/interactive design, media art, and the internet.


Introduction

Art 182 is an introductory computer programming course that explores the use of open source programming environments to create media based artwork. We will explore the history of media and computer based artworks as well as contemporary practitioners of the subject. Students will be introduced to the fundamental concepts of programming through the study of the Processing and p5.js programming environments. Assignments and readings will cover topics such as interactivity, typography, animation, and games. Classes will alternate between workshops / lectures and in class studio time. Studio, six hours. Outside Study, six hours.

Course Goals

1. Develop an understanding of the principles of programming.
2. Ability to use programming as a medium for creative work.
3. Awareness of the field, its pioneers and contemporary practitioners.
4. The ability to excel in future courses about the Internet, animation, interactive media, and game design.

Project 1, Part A -- Due Weds September 7

Code From class - 8/31


Design an abstract or simplified face composed of 10-20 basic shapes. Look at the links and images below as a reference. Create two emotions for this face, one neutral, and one exaggerated. For instance, a very sad frown, and a neutral expression. Have a plan in mind for how the face might transition from neutral to exaggerated. You may only move, rotate, and scale the facial features, it should not “pop” from one expression to the next. Think about all the different faces we’ve looked at and how you can construct a face both human or non-human.

Before you begin coding, sketch this face out either on paper, or in photoshop/illustrator/your software of choice and bring them to class on Wednesday 9/7. Also arrive with your neutral face translated into Processing code. Make sure your sketch size is set at 720x720px.

For Wednesday 9/7, also read the following chapters in the getting started with processing PDF. Pay special attention to The variables chapter!

Project 1, Part B -- Due Weds September 14

Add variables to your project from Part A so that the face can move between the neutral and exaggerated expressions by changing the value of the variables. The variables can change the color, position, scale, rotation, etc. of your face. Don't forget to write comments in your code so that you know the range of the variable i.e int x = 100; // left eye x - coordinate with a value between 100 and 200. Be careful when tring out rotation and scale, they can be tricky.

Bring this code to class on Monday September 12 to work on during in class studio time. One more reference for you! Mattias Dorfelt's Weird Faces

Project 1, Part C -- Due Mon September 26

Modify your code from part B so that the face changes fluidly from the neutral expression to the strong expression as you move the mouse across the screen. The map function will be your friend for this. You will need to include a setup() and draw() to your project so that variables in the draw() can be updated on the fly. Think about how you can develop in-between expressions and make the face as dynamic and emotive as possible. I have put together an example sketch for you to take a look at.

Bring your completed code to class on Monday Sept 26 ready to hand in.

For project 3 you will create an interactive drawing program a la Kid Pix / MS Paint / Brushes App for someone close to you. Craig Hickman created Kid Pix for his son Ben. Think about the person you are making the program for and their interests. What will engage and excite them? Based on that person's personality you will design and code a drawing application in Processing using images as rubber stamps. You need to make at least 5 but not more than 10 different tools / stamps.
